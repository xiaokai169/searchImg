## 三、核心架构 — 两阶段检索流程

```
用户上传图片
     │
     ▼
┌─────────────────────────────────────┐
│  特征提取 (DualModelExtractor)       │
│  ├─ CLIP ViT-B/32 → 512维向量 (语义) │
│  └─ ResNet50 → 2048维向量 (纹理)     │
└─────────────────────────────────────┘
     │
     ▼
┌─────────────────────────────────────┐
│  阶段1: CLIP粗召回 (FAISS内积索引)   │
│  ├─ 从全部向量中取 Top-200 候选       │
│  └─ 可选品类过滤                     │
└─────────────────────────────────────┘
     │
     ▼
┌─────────────────────────────────────┐
│  阶段2: ResNet50细粒度精排           │
│  ├─ 对200候选重算ResNet余弦相似度    │
│  ├─ 分数拉伸 [0.45, 1.0] → [0, 1]  │
│  ├─ 双阈值过滤 (MIN_RESNET≥0.60,     │
│  │   相对≥最高分×0.65)              │
│  └─ 返回 Top-20 结果                 │
└─────────────────────────────────────┘
```

### 关键技术决策

| 决策 | 说明 |
|------|------|
| **为什么用双模型？** | CLIP语义太宽泛（所有白底产品图都像），ResNet纹理区分度更好 |
| **为什么用ONNX？** | 无需PyTorch运行时，ONNX Runtime纯CPU推理，适合4核服务器 |
| **为什么用SQLite？** | 1万图片规模，零外部依赖，WAL模式支持读写并发 |
| **为什么用FAISS？** | Facebook的向量检索引擎，内积索引=余弦相似度，毫秒级检索 |

### 关键类说明

| 类/模块 | 职责 | 线程安全 |
|---------|------|----------|
| `TwoStageEngine` | FAISS索引管理 + 两阶段检索 | ✅ Lock |
| `DualModelExtractor` | CLIP + ResNet ONNX推理 | ✅ Lock |
| `LRUCache` | 搜索缓存(200条) | ✅ Lock |
| `database.py` | SQLite CRUD | ✅ thread-local |


## 七、数据存储详解

```
data/
├── images.db
│   └── images 表:
│       ├── id          INTEGER PRIMARY KEY AUTOINCREMENT
│       ├── faiss_id    INTEGER NOT NULL UNIQUE    # FAISS索引内部ID
│       ├── filename    TEXT NOT NULL               # 文件名
│       ├── category    TEXT NOT NULL DEFAULT '其他' # 品类
│       ├── file_size   INTEGER DEFAULT 0           # 文件大小
│       ├── width       INTEGER DEFAULT 0           # 图片宽度
│       ├── height      INTEGER DEFAULT 0           # 图片高度
│       └── create_time TIMESTAMP DEFAULT CURRENT_TIMESTAMP
│       索引: idx_images_category, idx_images_faiss_id, idx_images_create_time
│
├── faiss_clip_index.bin       # FAISS IndexIDMap(IndexFlatIP) - CLIP 512维
├── resnet_features.npy        # (N, 2048) float32 - ResNet特征矩阵
├── resnet_features_ids.npy    # (N,) int64 - faiss_id 顺序映射
├── id_map.json                # {"next_id": N} - ID分配计数器
└── index_progress.json        # 索引任务进度 {"status":"idle","total":0,...}
```

### 存储策略

- **SQLite WAL模式**：允许读写并发，`synchronous=NORMAL` 性能优先
- **FAISS IndexIDMap**：支持自定义ID映射，内积索引 = 余弦相似度
- **ResNet特征分离存储**：不放入FAISS（检索时才计算），节省内存
- **LRU缓存**：200条搜索缓存，相同图片重复查询秒级返回

## 八、注意事项

### 安全

1. **`sync_products.py` 的 JWT Token** 已改为从环境变量 `ARAB_BEE_TOKEN` 读取，运行前需先设置：
   ```bash
   export ARAB_BEE_TOKEN="your_jwt_token_here"
   ```

### 模型

2. **ONNX 模型已就绪**：`models/` 目录下已有 CLIP (~336MB) 和 ResNet50 (~90MB) 两个模型文件，无需重新下载。若需重新生成，运行 `python download_models.py`（需要 PyTorch + open_clip_torch + torchvision）。

### 运行

3. **线程安全**：所有核心组件（Engine、Extractor、Cache）都使用 `threading.Lock` 实现了线程安全，支持 Flask 多线程模式。
4. **优雅关闭**：`app.py` 注册了 `atexit` 和信号处理（SIGINT/SIGTERM），退出时自动保存 FAISS 索引和 ResNet 特征到磁盘。
5. **首次启动**：如果 `data/` 目录下没有索引文件，系统会自动创建空索引，通过 API 或导入脚本添加图片后即可搜索。
